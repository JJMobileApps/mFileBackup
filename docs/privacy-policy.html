한국어

1. 개요

mFileBackup(이하 "앱")은 사용자가 자신의 기기 또는 별도로 보유한 기기를 백업 서버로 구성하여 사진, 동영상, 파일 및 관련 메타데이터를 백업, 조회, 전송, 관리할 수 있도록 돕는 애플리케이션입니다. 본 개인정보처리방침은 앱이 접근, 수집, 저장, 전송, 이용할 수 있는 정보와 그 목적을 설명합니다.

2. 앱이 접근하거나 처리할 수 있는 정보

  • 사용자 정보: 이름, 아이디, 비밀번호와 같은 앱 내부 사용자 계정 정보
  • 연결 정보: 서버 주소, 포트, 로컬 네트워크 연결 정보, 기기 식별에 필요한 정보
  • 파일 및 미디어 정보: 사진, 동영상, 문서, 폴더 경로, 파일명, 크기, 수정 시각, 미리보기 생성에 필요한 정보
  • 메타데이터: 백업 상태, 즐겨찾기, 공유 설정, 갤러리 분류 정보, 백업 예약 정보
  • 로그 및 진단 정보: 오류 로그, 동기화 상태, 보안 이벤트, 접속 이력, 앱 동작 기록
  • 결제 관련 정보: 프리미엄 기능 구매 여부 확인을 위한 Google Play 결제 상태 정보
  • 알림 및 일정 정보: 자동 백업 알림, 백그라운드 작업, 부팅 후 재시작에 필요한 정보

3. 정보 이용 목적

  • 백업 서버 구성 및 클라이언트 연결 제공
  • 사진, 동영상, 파일의 업로드, 다운로드, 동기화, 조회 기능 제공
  • 사용자 인증, 접근 제어, 승인 기기 관리, 로그인 보안 기능 제공
  • 자동 백업 예약, 갤러리 조회, 즐겨찾기 및 공유 기능 제공
  • 오류 확인, 품질 개선, 장애 대응, 보안 기록 관리
  • 프리미엄 기능 구매 상태 확인, 복원, 제한 기능 제어

4. 정보 저장 및 전송 방식

앱은 주로 사용자의 기기 또는 사용자가 직접 구성한 서버 기기 내부에 데이터를 저장합니다. 사용자가 서버/클라이언트 기능을 사용할 경우, 필요한 데이터는 사용자의 기기들 사이 또는 사용자가 지정한 네트워크 경로를 통해 전송될 수 있습니다.

개발자는 별도의 중앙 클라우드 서버를 운영하여 사용자의 백업 파일 전체를 수집하는 것을 기본 동작으로 하지 않습니다. 다만, Google Play 결제 기능 사용 시 구매 상태 확인과 같은 일부 정보 처리는 Google Play 결제 시스템을 통해 이루어질 수 있습니다.

5. 제3자 제공 및 공유

  • 앱은 사용자가 요청한 서버/클라이언트 연결 및 공유 기능 수행을 위해 사용자의 다른 기기 또는 다른 사용자와 데이터를 공유할 수 있습니다.
  • 인앱결제 및 구매 복원 기능은 Google Play 결제 시스템을 사용합니다.
  • 법령에 따른 요청이 있는 경우 관련 법령에 따라 필요한 범위 내에서 정보를 제공할 수 있습니다.

6. 권한 사용

앱은 다음과 같은 권한을 사용할 수 있습니다.

  • 저장소 및 미디어 권한: 사진, 동영상, 파일 백업 및 조회
  • 네트워크 권한: 서버 연결, 파일 전송, 동기화
  • 알림 권한: 백업 진행 상태 및 중요 알림 표시
  • 정확한 알람, 부팅 완료, 배터리 최적화 예외 관련 권한: 자동 백업 예약 유지
  • 모든 파일 접근 권한이 필요한 경우: 사용자가 선택한 외부 저장소 또는 전체 파일 백업 기능 제공

7. 보관 및 삭제

앱이 저장하는 정보는 기본적으로 사용자의 기기 또는 사용자가 직접 운영하는 서버 기기에 보관됩니다. 사용자는 앱 기능을 통해 사용자 계정, 백업 프로필, 즐겨찾기, 공유 정보, 파일 및 일부 로그를 삭제할 수 있습니다. 앱 삭제, 데이터 초기화, 또는 서버 데이터 정리를 통해 저장 정보가 제거될 수 있습니다.

8. 보안

앱은 인증, 접근 제한, 승인 기기 관리, 로그인 제한, 로컬 저장, 네트워크 연결 제어 등 기능을 통해 사용자 데이터를 보호하려고 노력합니다. 다만, 사용자가 구성한 기기 환경, 네트워크 환경, 운영체제 설정에 따라 실제 보안 수준은 달라질 수 있습니다.

9. 아동의 개인정보

앱은 일반적인 파일 백업 및 개인 서버 운영 용도를 대상으로 하며, 아동을 주요 대상으로 설계되지 않았습니다.

10. 정책 변경

본 개인정보처리방침은 앱 기능 변경, 법령 변경, 서비스 운영 변경에 따라 수정될 수 있습니다. 중요한 변경이 있는 경우 앱, 웹페이지 또는 스토어 등록정보를 통해 고지할 수 있습니다.

English

1. Overview

mFileBackup ("the App") helps users configure their own device, or another device they own, as a backup server for photos, videos, files, and related metadata. This Privacy Policy explains what information the App may access, collect, store, transmit, and use, and for what purposes.

2. Information the App may access or process

  • User information: in-app account information such as display name, user ID, and password
  • Connection information: server address, port, local network connection details, and device identification data
  • File and media information: photos, videos, documents, folder paths, file names, file sizes, modified times, and preview-related data
  • Metadata: backup status, favorites, sharing settings, gallery categorization, and scheduled backup settings
  • Log and diagnostic information: error logs, sync status, security events, connection history, and operational records
  • Billing information: Google Play purchase status information used to verify premium entitlement
  • Notification and scheduling information: data needed for scheduled backups, foreground tasks, and restart-after-boot behavior

3. How the information is used

  • To provide backup server configuration and client connectivity
  • To provide upload, download, synchronization, browsing, and gallery features for files and media
  • To provide user authentication, access control, approved device management, and login security features
  • To provide scheduled backups, favorites, and sharing features
  • To identify errors, improve quality, respond to failures, and manage security records
  • To verify, restore, and enforce premium purchase status

4. Storage and transmission

The App primarily stores data on the user's device or on a server device directly configured by the user. When server or client features are used, required data may be transmitted between the user's devices or through network paths explicitly configured by the user.

The developer does not operate a central cloud service that collects all user backup files as the default service behavior. However, when Google Play Billing is used, limited purchase-related processing may occur through Google Play's billing systems.

5. Sharing with third parties

  • The App may share data with the user's other devices, or with other users, when the user explicitly uses server/client or sharing features.
  • In-app purchases and purchase restoration are handled through Google Play Billing.
  • Information may be disclosed if required by applicable law or a lawful request.

6. Permissions

The App may request and use the following permissions:

  • Storage and media permissions: to back up and browse photos, videos, and files
  • Network permissions: to connect to the server, transfer files, and synchronize data
  • Notification permission: to display backup progress and important notices
  • Exact alarm, boot completed, and battery optimization exception-related permissions: to maintain scheduled backups
  • All files access, where required: to support user-selected external storage and broader file backup features

7. Retention and deletion

Information handled by the App is generally retained on the user's device or the user-managed server device. Users may delete in-app user accounts, backup profiles, favorites, sharing information, files, and some logs through the App's features. Information may also be removed by uninstalling the App, clearing app data, or manually deleting server-side content.

8. Security

The App uses features such as authentication, access restrictions, approved device management, login controls, local storage, and network configuration options to help protect user data. Actual security depends in part on the user's device configuration, network environment, and operating system settings.

9. Children's privacy

The App is intended for general file backup and personal server use and is not designed primarily for children.

10. Changes to this policy

This Privacy Policy may be updated to reflect changes in app features, legal requirements, or service operations. Important changes may be announced through the App, this webpage, or the store listing.
